## Why Split Planning From Research?

Trying to make a single agent perform planning, search execution, fact extraction, and answer synthesis simultaneously causes multiple critical failure modes:

1. **Context Bloat & Task Drift**: As an agent searches the web, retrieves pages, and tries to summarize them in a single reasoning window, the prompt context grows rapidly. The agent easily loses track of its original goal, starts chasing irrelevant links, or forgets to address key aspects of the user's query.
2. **Premature Convergence**: A single agent frequently makes a single web search, finds one partial answer, and decides it is "done" without checking if other perspectives or facts are missing.
3. **Redundant Iteration**: Lacking a pre-configured plan, the agent may execute the same search query multiple times with minor word changes, wasting API tokens and execution time.

### How Our Split Solves This
By isolating the **Planner** from the **Researcher**:
- The **Planner** focuses exclusively on deconstructing the user's high-level request into discrete, manageable sub-queries while the context window is clean.
- The **Researcher** executes targeted web searches sequentially for each isolated question, ensuring deep coverage.
- A final **Synthesis** task integrates the individual sub-results into a coherent response, without having its reasoning polluted by raw HTML scrapings.

## What Is a State Graph?

A **State Graph** is a design pattern for orchestrating workflows, structured around:
- **Shared State**: A central, validated memory store accessible to all nodes.
- **Nodes**: Independent Python functions that receive the current state, execute actions, and return updates to be merged back into the state.
- **Edges**: Directed lines dictating the execution route between nodes (either deterministic or conditional).

### State Graph in Our Codebase
We compile our workflow in [graph.py](file:///home/pulkit/projects/multi-agent%20reseach%20assistant/backend/app/graph.py) using the following parameters:

#### 1. The Shared State (`GraphState`)
A Pydantic model maintaining these memory fields:
- `original_query` (str): Stores the initial raw request input by the user.
- `plan` (List[str]): Stores the list of 1-4 sub-questions generated by the planner.
- `research_results` (List[dict]): Holds individual results from querying the research agent on each sub-question.
- `final_answer` (str): Holds the final compiled text answer.
- `sources` (List[str]): Holds unique URLs cited.

#### 2. The Nodes
- **`planner` node**: Takes `original_query`, invokes `PlannerAgent.plan()`, and outputs `{"plan": sub_questions}` to update the state.
- **`research` node**: Reads `plan` from the state, runs the `ResearchAgent` sequentially on each sub-question, aggregates answers into `research_results`, calls the LLM to synthesize the final answer, and writes `final_answer` and `sources` updates.

#### 3. The Ededges
- `START -> "planner"`: Initiates execution.
- `"planner" -> "research"`: Passes the generated plan to the researcher.
- `"research" -> END`: Terminates execution and returns final state values.

## Questions I Still Have
- **Question**: Explain in brief how does LangGraph work with some simple analogies.
- **Answer**: 
  Think of a LangGraph `StateGraph` as a **factory assembly line**:
  1. **The Shared State (`GraphState`)** is a **conveyor belt carrying a clipboard**. As the clipboard travels down the belt, workers read from it and write new information onto it.
  2. **The Nodes** are **specialized workers** standing along the belt. Each worker has a specific task (e.g., the *Planner* worker writes down research questions, the *Researcher* worker goes to look up web results, etc.). They take the clipboard, do their work, update the clipboard, and send it back.
  3. **The Edges** are **routing tracks**. They guide where the clipboard goes next. Normal edges are straight tracks, while **conditional edges** act as a routing inspector who looks at the clipboard content and decides which worker to route it to next (e.g. "if the research isn't complete, route it back to search").
